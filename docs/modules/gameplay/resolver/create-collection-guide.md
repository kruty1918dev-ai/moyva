# Resolver - Гайд створення нової колекції

## Мета

Практично створити нову колекцію, додати типи, варіації, prefab/sprite, перевірити інтеграцію з підбірником.

<details>
<summary>Крок 1. Відкрити потрібний інструмент</summary>

1. Відкрийте Moyva/Construction/Wall Registry Editor.
2. Розкрийте блок "Вбудований модуль резолвера".
3. Переконайтесь, що призначено BuildingRegistrySO.

Чому це потрібно:

- саме тут редагується canonical-конфігурація case -> variants.

</details>

<details>
<summary>Крок 2. Створити колекцію</summary>

1. Натисніть + Додати колекцію у Wall Registry.
2. Заповніть CollectionId, WallBuildingId, GateBuildingId.

Що відбувається:

- створюється контейнер для топологічних типів і воріт.

</details>

<details>
<summary>Крок 3. Додати типи через '+' у резолвері</summary>

1. У блоці резолвера біля потрібної колекції натисніть +.
2. Оберіть тип зі списку (corner/T/cross/end/diagonal...).
3. Прочитайте tooltip перед вибором (там опис контексту і призначення).

Альтернатива:

- Якщо частина типів не потрібна, не додавайте їх: система підтримує часткові реалізації.

</details>

<details>
<summary>Крок 4. Додати реалізацію типу</summary>

Варіант A (готовий prefab):

1. Заповніть Variant ID.
2. Передайте prefab.
3. Натисніть Додати.

Варіант B (лише sprite):

1. Заповніть Variant ID (або залиште для авто-ID).
2. Передайте sprite.
3. Натисніть Додати.

Що виконується автоматично:

- генерація prefab з sprite;
- реєстрація/оновлення BuildingDefinition;
- додавання ID у TopologyBindings.

</details>

<details>
<summary>Крок 5. Налаштування варіацій</summary>

1. Для того ж типу додайте ще 1..N варіантів ID.
2. Залиште 1 варіант, якщо рандом не потрібен.

Правило runtime:

- 1 варіант -> вибирається завжди;
- 2+ варіанти -> вибір випадковий.

</details>

<details>
<summary>Крок 6. Перевірка результату</summary>

1. Запустіть сцену і поставте елементи в різних конфігураціях (лінія, кут, Т, хрест, кінці).
2. Перевірте, що підбірник повертає очікувані ID для кожного кейсу.
3. Звірте лічильник реалізовано X/22 у резолвері.

</details>

<details>
<summary>Типові помилки і виправлення</summary>

- Тип є, але не працює: перевірте, що у типі є хоча б один непорожній Variant ID.
- Переданий sprite, але prefab не створився: перевірте шлях папки в полі auto-prefab і права на запис в Assets.
- Неправильний кут: перевірте семантику атласу для кутів (візуальна позиція vs логічні конектори).

</details>
