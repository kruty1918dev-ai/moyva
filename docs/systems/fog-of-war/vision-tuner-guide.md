# Fog of War — Vision Tuner Guide

← [README](README.md)

---

## Що це

`Vision Tuner` — окремий editor-інструмент для балансу видимості без Play Mode.

Відкриття:

`Moyva/Tools/Fog of War/Vision Tuner`

---

## Можливості

| Секція | Для чого |
|---|---|
| `Assets` | Призначення `FogOfWarSettings` та `UnitRegistrySO`, авто-пошук assets |
| `FogOfWarSettings` | Тюнінг глобальних параметрів зору та висотних правил, reset і застосування preset |
| `Unit Vision Ranges` | Редагування `VisionRange` для кожного типу юніта |
| `Height Vision Preview` | Швидка перевірка формул дальності і line-of-sight |

---

## Presets і Reset

У секції `FogOfWarSettings` доступні:

1. `Preset` selector з 10 готовими профілями.
2. `Apply Preset` для швидкого заповнення значень.
3. `Reset Settings` для повернення до базового профілю `01 Default`.

Після застосування будь-якого пресету всі значення залишаються повністю редагованими вручну.

### Список пресетів

| Пресет | Призначення |
|---|---|
| `01 Default` | Базовий збалансований профіль |
| `02 Flat Map` | Майже рівна мапа зі слабким впливом висоти |
| `03 Gentle Hills` | М'які пагорби з помірною роллю висоти |
| `04 Rugged Hills` | Більш відчутний вплив рельєфу без крайнощів |
| `05 Mountain Warfare` | Гірський профіль з сильними бонусами/штрафами |
| `06 Scout Friendly` | Профіль для активної розвідки і довшого огляду |
| `07 Defensive Ridge` | Оборонна перевага висот і хребтів |
| `08 Harsh Uphill` | Жорсткий penalty для погляду вгору |
| `09 Long Sightlines` | Відкриті простори з дальнім оглядом |
| `10 Close Quarters` | Компактна видимість для щільних мап |

## Параметри з тултіпами

### FogOfWarSettings

| Параметр | Що означає |
|---|---|
| `Default Vision Range` | Fallback радіус, якщо юніт не передав власний `VisionRange` |
| `Min Vision Range` | Глобальний мінімум радіуса (поточна модель очікує не менше `1`) |
| `Max Vision Range` | Глобальний максимум ефективного радіуса після бонусів/штрафів |
| `Elevation Step` | Крок різниці висоти для нарахування бонусу або штрафу |
| `Observer Height Bonus / Step` | Бонус за кожен крок висоти тайлу спостерігача |
| `Downhill Bonus / Step` | Бонус за погляд вниз по схилу |
| `Uphill Penalty / Step` | Штраф за погляд вгору по схилу |
| `Max Observer Bonus` | Стеля бонусу від висоти спостерігача |
| `Max Downhill Bonus` | Стеля бонусу за погляд вниз |
| `Max Uphill Penalty` | Стеля штрафу за погляд вгору |
| `Occlusion Slope Bias` | Допуск для line-of-sight перевірки проти дрібних перепадів |

### Unit Vision Ranges

| Параметр | Що означає |
|---|---|
| `Type` | `UnitClassConfig.TypeId` (тільки читання) |
| `Vision` | Базовий радіус для конкретного типу юніта (clamp до мінімуму `1`) |

### Height Vision Preview

| Параметр | Що означає |
|---|---|
| `Base Vision Range` | Базова дальність до застосування модифікаторів рельєфу |
| `Target Distance (Chebyshev)` | Відстань до цілі у метриці `max(|dx|, |dy|)` |
| `Observer Height` | Висота тайлу спостерігача |
| `Target Height` | Висота тайлу цілі |
| `Use Blocker Sample` | Увімкнення перевірки line-of-sight через проміжну точку |
| `Blocker Height` | Висота проміжної точки на промені |
| `Blocker Distance` | Відстань до проміжної точки від спостерігача |

## Рекомендований workflow

1. Призначте `FogOfWarSettings` і `UnitRegistrySO` у секції `Assets`.
2. Оберіть один із 10 пресетів як стартову точку.
3. Натисніть `Apply Preset`.
4. За потреби вручну підкоригуйте Height Vision параметри.
5. Перевірте сценарії у `Height Vision Preview`.
6. Підкоригуйте `Vision` для окремих типів юнітів.
7. Якщо треба повернутися до базового профілю, використайте `Reset Settings`.
8. Запустіть сцену та перевірте реальну поведінку на мапі.

---

## Нотатки

- Параметри вікна змінюють assets напряму через `SerializedObject`.
- `Vision` для юнітів автоматично затискається до `>= 1`.
- Якщо `TypeId` містить `_`, вікно покаже попередження про рекомендований формат ID.
