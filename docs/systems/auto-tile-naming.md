# Система найменування тайлів для Auto-Tile

## Сітка 3×3 — орієнтація напрямків

Кожен тайл аналізується відносно **центрального тайлу** (🎯).  
Сусіди визначаються у **8 напрямках** (4 кардинальні + 4 діагональні):

```
         Стовпець
       x-1   x   x+1
      ┌─────┬─────┬─────┐
y+1   │ NW  │  N  │ NE  │  ← рядок вище
      ├─────┼─────┼─────┤
y     │  W  │ 🎯  │  E  │  ← поточний рядок
      ├─────┼─────┼─────┤
y-1   │ SW  │  S  │ SE  │  ← рядок нижче
      └─────┴─────┴─────┘
```

### Що означає кожен напрямок

| Напрямок | Позиція відносно центру | Зміщення (dx, dy) |
|----------|-------------------------|-------------------|
| **N**  (Північ)          | Зверху по центру     | (0, +1)  |
| **NE** (Північний Схід)  | Зверху справа        | (+1, +1) |
| **E**  (Схід)            | Справа по центру     | (+1, 0)  |
| **SE** (Південний Схід)  | Знизу справа         | (+1, -1) |
| **S**  (Південь)         | Знизу по центру      | (0, -1)  |
| **SW** (Південний Захід) | Знизу зліва          | (-1, -1) |
| **W**  (Захід)           | Зліва по центру      | (-1, 0)  |
| **NW** (Північний Захід) | Зверху зліва         | (-1, +1) |

> **Примітка**: «Зверху» означає y+1, «Знизу» означає y-1. Це Unity-координати.

---

## Формат ID тайлу

```
{базовий-тип}[-{висота}]-{суфікс}
```

Роздільник — завжди **дефіс** `-`. Символ `_` заборонено.

Приклади:
- `grass` — чистий тайл без переходів
- `grass-cliff-N` — трава з обривом на півночі
- `grass-2-cliff-NE` — трава, висота 2, обрив на Пн-Сх
- `water-corner-NE` — вода з кутом на Пн-Сх

## Система 1: AutoTileTransitionNode (обриви і кути)

Ця система генерує **cliff** та **corner** суфікси, аналізуючи яким тайлам сусіди відрізняються від центрального.

### Як це працює

1. Береться центральний тайл (наприклад `grass`)
2. Перевіряються всі 8 сусідів
3. Якщо сусід має **інший базовий тип** — це «край» (edge)
4. На основі кількості та розташування країв визначається суфікс

### Cliff-суфікси (кардинальні краї)

Якщо **хоча б одна кардинальна сторона** (N/E/S/W) — край:

#### 1 край — одна стіна

```
┌─────┬─────┬─────┐        ┌─────┬─────┬─────┐
│     │ ▓▓▓ │     │        │     │     │     │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│     │ 🎯  │     │        │     │ 🎯  │ ▓▓▓ │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│     │     │     │        │     │     │     │
└─────┴─────┴─────┘        └─────┴─────┴─────┘
     cliff-N                    cliff-E
```

| Суфікс | Де інший тайл | Візуально |
|--------|--------------|-----------|
| `cliff-N` | Зверху (Північ) | Стіна зверху |
| `cliff-E` | Справа (Схід)   | Стіна справа |
| `cliff-S` | Знизу (Південь)  | Стіна знизу |
| `cliff-W` | Зліва (Захід)    | Стіна зліва |

#### 2 краї — суміжні

```
┌─────┬─────┬─────┐        ┌─────┬─────┬─────┐
│     │ ▓▓▓ │     │        │     │ ▓▓▓ │     │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│     │ 🎯  │ ▓▓▓ │        │     │ 🎯  │     │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│     │     │     │        │     │ ▓▓▓ │     │
└─────┴─────┴─────┘        └─────┴─────┴─────┘
     cliff-NE                   cliff-NS
```

| Суфікс | Стіни | Опис |
|--------|-------|------|
| `cliff-NE` | Зверху + Справа   | Кут: Пн-Сх |
| `cliff-NW` | Зверху + Зліва    | Кут: Пн-Зх |
| `cliff-SE` | Знизу + Справа    | Кут: Пд-Сх |
| `cliff-SW` | Знизу + Зліва     | Кут: Пд-Зх |
| `cliff-NS` | Зверху + Знизу    | Протилежні: вертикальна щілина |
| `cliff-EW` | Справа + Зліва    | Протилежні: горизонтальна щілина |

#### 3 краї — відкрита одна сторона

```
┌─────┬─────┬─────┐
│     │ ▓▓▓ │     │    cliff-NSW: Відкрита тільки Північ
├─────┼─────┼─────┤    Стіна зверху НЕМАЄ, а Пд+Зх+Сх — є
│ ▓▓▓ │ 🎯  │     │
├─────┼─────┼─────┤
│     │ ▓▓▓ │     │
└─────┴─────┴─────┘
```

| Суфікс | Яка сторона ВІДКРИТА | Стіни на 3 сторонах |
|--------|---------------------|---------------------|
| `cliff-SEW` | Відкрита **Північ** | Пд + Сх + Зх |
| `cliff-NSW` | Відкритий **Схід**  | Пн + Пд + Зх |
| `cliff-NEW` | Відкритий **Південь** | Пн + Сх + Зх |
| `cliff-NSE` | Відкритий **Захід** | Пн + Пд + Сх |

> **Увага**: Назва `cliff-SEW` означає стіни на S+E+W, тобто **відкрита тільки N**.

#### 4 краї — повне оточення

| Суфікс | Опис |
|--------|------|
| `cliff-ALL` | Усі 4 кардинальні сторони — інший тайл. Ізольований тайл. |

### Corner-суфікси (діагональні кути)

Генеруються **тільки якщо кардинальні сторони однакові** (немає cliff), але **діагональні** сусіди відрізняються.

```
┌─────┬─────┬─────┐
│ ▓▓▓ │ === │     │    corner-NW: кардинальних країв немає
├─────┼─────┼─────┤    але NW-сусід (зверху-зліва) — інший тайл
│ === │ 🎯  │ === │
├─────┼─────┼─────┤
│     │ === │     │
└─────┴─────┴─────┘
  === означає такий самий тайл як центр
```

| Суфікс | Де діагональний край |
|--------|---------------------|
| `corner-NE` | Зверху справа |
| `corner-SE` | Знизу справа |
| `corner-SW` | Знизу зліва |
| `corner-NW` | Зверху зліва |

Кути комбінуються:
- `corner-NE-SE` — два кути справа
- `corner-NE-SW-NW` — три кути
- `corner-NE-SE-SW-NW` — усі 4 кути

**Порядок у назві завжди**: NE → SE → SW → NW

## Система 2: ObjectAutoTileNode (лінійні об'єкти — річки, дороги)

Аналізує лише **4 кардинальні напрямки** (без діагоналей) для об'єктів типу річок та доріг.

### 4-бітова маска

```
Біт 0 (значення 1):  N — сусід зверху
Біт 1 (значення 2):  E — сусід справа
Біт 2 (значення 4):  S — сусід знизу
Біт 3 (значення 8):  W — сусід зліва
```

### Повна таблиця варіантів (16 комбінацій)

| Маска | Сусіди | ID-суфікс | Візуалізація | Опис |
|:-----:|--------|-----------|:------------:|------|
| 0 | — | *(fallback)* | `·` | Ізольований |
| 1 | N | `-end-n` | `↑` | Тупик, вихід на Північ |
| 2 | E | `-end-e` | `→` | Тупик, вихід на Схід |
| 3 | N+E | `-corner-ne` | `└` | Кут: з Пн на Сх |
| 4 | S | `-end-s` | `↓` | Тупик, вихід на Південь |
| 5 | N+S | `-vertical` | `│` | Вертикальна лінія |
| 6 | E+S | `-corner-se` | `┌` | Кут: зі Сх на Пд |
| 7 | N+E+S | `-t-east` | `├` | Т-розгалуження на Схід |
| 8 | W | `-end-w` | `←` | Тупик, вихід на Захід |
| 9 | N+W | `-corner-nw` | `┘` | Кут: з Пн на Зх |
| 10 | E+W | `-horizontal` | `─` | Горизонтальна лінія |
| 11 | N+E+W | `-t-north` | `┴` | Т-розгалуження на Північ |
| 12 | S+W | `-corner-sw` | `┐` | Кут: з Пд на Зх |
| 13 | N+S+W | `-t-west` | `┤` | Т-розгалуження на Захід |
| 14 | E+S+W | `-t-south` | `┬` | Т-розгалуження на Південь |
| 15 | N+E+S+W | `-cross` | `┼` | Хрещення (4 боки) |

> **Легенда**: ▓▓▓ = сусідній тайл того ж типу (з'єднання є), пробіл = інший тип (з'єднання немає).

### Ілюстрації — Т-розгалуження (t-junction)

```
┌─────┬─────┬─────┐        ┌─────┬─────┬─────┐
│     │ ▓▓▓ │     │        │     │ ▓▓▓ │     │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│ ▓▓▓ │ 🎯  │ ▓▓▓ │        │     │ 🎯  │ ▓▓▓ │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│     │     │     │        │     │ ▓▓▓ │     │
└─────┴─────┴─────┘        └─────┴─────┴─────┘
  t-north (N+E+W)             t-east (N+E+S)
  відкритий Південь           відкритий Захід

┌─────┬─────┬─────┐        ┌─────┬─────┬─────┐
│     │     │     │        │     │ ▓▓▓ │     │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│ ▓▓▓ │ 🎯  │ ▓▓▓ │        │ ▓▓▓ │ 🎯  │     │
├─────┼─────┼─────┤        ├─────┼─────┼─────┤
│     │ ▓▓▓ │     │        │     │ ▓▓▓ │     │
└─────┴─────┴─────┘        └─────┴─────┴─────┘
  t-south (E+S+W)             t-west (N+S+W)
  відкрита Північ             відкритий Схід
```

> **Підказка для t-junction**: назва містить **напрямок відгалуження** (куди є 3-й вихід), а не відкриту сторону.  
> `t-east` = основна вісь N↔S + відгалуження на **E** (Схід).

### Приклади ID для річки

```
river              ← ізольований (fallback)
```

## Висотні рівні (Height Tiers)

Якщо підключена heightmap, додається номер рівня **перед** суфіксом:

| Висота h | Рівень | Приклад |
|----------|--------|---------|
| h < 0.2  | 0 (не додається) | `grass-cliff-N` |
| 0.2 ≤ h < 0.4 | 1 | `grass-1-cliff-N` |
| 0.4 ≤ h < 0.6 | 2 | `grass-2-cliff-N` |
| 0.6 ≤ h < 0.8 | 3 | `grass-3-cliff-N` |
| h ≥ 0.8 | 4 | `grass-4-cliff-N` |

## Чекліст: мінімальні тайли для одного типу

Якщо ви створюєте новий тип тайлу (наприклад `sand`) і хочете повну підтримку авто-підбору:

- [ ] `sand` — базовий тайл (без суфікса)
- [ ] 15 cliff-варіантів
- [ ] 15 corner-варіантів
- [ ] **Разом: 31 запис у TileRegistrySO**

Для об'єктів (річки/дороги):
- [ ] `river` — базовий (fallback)
- [ ] 15 варіантів від масок
- [ ] **Разом: 16 записів у MapObjectRegistrySO**

> Відсутні тайли не зламають генерацію — просто будуть використані базові. Але візуально переходи виглядатимуть грубо.
