# Tutorial: Як додати новий ресурс

Ціль: додати ресурс так, щоб він коректно працював у редакторі, runtime і UI.

## Передумови

- Відкрито Economy Hub.
- Призначено EconomyDatabaseSO.
- Призначено RulesConfig і RulesTemplate.

## Крок 1. Створити ресурсний asset

1. Перейдіть у вкладку Ресурси.
2. Натисніть Створити.
3. Задайте ім'я ассета (приклад: WheatResource).

Результат: з'являється EconomyResourceDefinition asset.

## Крок 2. Заповнити поля ресурсу

1. Вкажіть Id (приклад: wheat).
2. Вкажіть DisplayName (приклад: Пшениця).
3. Вкажіть Category:
- Food для їжі;
- Materials для матеріалів.
4. За потреби додайте іконку та інші візуальні поля.

Важливо: Id має бути унікальним у всій базі.

## Крок 3. Додати ресурс у базу

1. Переконайтеся, що asset додано до списку Resources у EconomyDatabaseSO.
2. Якщо asset існує, але не в базі, використайте Додати/вибрати існуючий.

Результат: runtime зможе знаходити ресурс за id.

## Крок 4. Підв'язати ресурс у виробництво/склади

1. У Production профілях додайте id ресурсу як input або output.
2. У Warehouse policy перевірте, що ресурс дозволений/пріоритизований за потреби.

Результат: ресурс реально бере участь в економічному циклі.

## Крок 5. Перевірити правила і формати

1. В Rules переконайтеся, що загальні параметри валідні.
2. Для UI-підсумків перевірте параметри:
- ui-summary-materials-format
- ui-summary-food-format

Нотатка: самі формати знаходяться в RulesTemplate, не в UI-компонентах.

## Крок 6. Запустити валідацію

1. Відкрийте вкладку Валідація.
2. Натисніть Run Validation.
3. Усуньте Error; Warning перевірте вручну.

## Крок 7. Прогнати симуляцію

1. Перейдіть у вкладку Симуляція.
2. Оберіть settlement і тривалість (наприклад 10 хв).
3. Запустіть deterministic preview.
4. Перевірте, що новий ресурс з'являється в очікуваних підсумках.

## Крок 8. Runtime перевірка в сцені

1. Запустіть сцену з EconomyInstaller.
2. Розмістіть будівлі, які працюють з новим ресурсом.
3. Перевірте сигнали/лог і зміну пулу ресурсів.
4. Перевірте summary UI (Food/Materials) для активного owner.

## Крок 9. Мультиплеєр/бот перевірка

1. Задайте різні owner у будівництві.
2. Перевірте, що суми ресурсів не змішуються між owner.
3. Переконайтеся, що UI кожного owner показує лише його підсумок.

## Часті помилки

- Дубль Id ресурсу.
- Ресурс створений як asset, але не доданий у EconomyDatabaseSO.
- Неправильна Category (Food замість Materials і навпаки).
- Профіль виробництва посилається на неіснуючий resourceId.
- Пропущена валідація перед комітом.
