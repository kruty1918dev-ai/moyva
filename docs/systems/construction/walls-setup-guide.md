## 1. Як це працює — коротко

Стіна в грі — це не одна картинка. Залежно від того, з якого боку стоять сусіди, система автоматично підбирає правильний варіант сегмента:

```
      NE-кут   Вертикальна   NW-кут
         ┐          │          ┌
         │    ←——————————→    │
Горизонт ─         (нема)      ─ Горизонт

         └          │          ┘
      SE-кут   Вертикальна   SW-кут
```

Всього **6 варіантів сегмента стіни** + **1 варіант воріт**. Це і є одна **колекція стін**.

Можна мати кілька колекцій (кам'яна стіна, дерев'яна стіна, замкова стіна) — кожна має свій набір prefab'ів.

---

## 2. Що потрібно підготувати

Перед початком переконайтесь:

| Що | Де знайти |
|---|---|
| Unity відкритий на потрібній сцені | — |
| `BuildingRegistrySO` вже є в проєкті | `Assets/Moyva/Settings/` або шукайте через `Project → Moyva/Construction/BuildingRegistry` |
| Є 6 sprites/textures для варіантів стіни та 1 для воріт | Підготовлені арт-ресурси або тимчасові заглушки |

> **Немає sprites? Не страшно.** Для тестування можна використати Unity-примітиви (квадрат одного кольору). Замінити на справжній арт завжди можна пізніше.

## Крок 1 — Створення prefab'ів стін і воріт

Для кожного варіанта стіни потрібен окремий prefab. Разом: **6 стін + 1 ворота = 7 prefab'ів**.

### 1.1 Структура папок

Рекомендована структура:

```
Assets/
└── Moyva/
    └── Prefabs/
        └── Buildings/
            └── Walls/
                ├── Wall_Horizontal.prefab
                ├── Wall_Vertical.prefab
                ├── Wall_Corner_NE.prefab
                ├── Wall_Corner_NW.prefab
                ├── Wall_Corner_SE.prefab
                ├── Wall_Corner_SW.prefab
                └── Gate.prefab
```

### 1.2 Як створити один prefab стіни

1. У **Hierarchy** Right-click → `2D Object → Sprite`.
2. Назвіть GameObject, наприклад `Wall_Horizontal`.
3. У компоненті `Sprite Renderer`:
   - `Sprite` — призначте потрібний sprite для горизонтальної стіни.
   - `Order in Layer` — встановіть значення вище за тайловий шар (наприклад, `2`).
4. Впевніться, що pivot sprite відцентрований (`Center`), щоб стіна ставала на тайл рівно.
5. У **Project** window знайдіть свій `Assets/Moyva/Prefabs/Buildings/Walls/` і **перетягніть** GameObject туди — так створиться prefab.
6. Видаліть GameObject зі сцени (він більше не потрібен в ієрархії).

> **Повторіть** пункти 1.2 для всіх 7 варіантів (горизонтальна, вертикальна, 4 кути, ворота).

### 1.3 Орієнтація sprites

Система **не крутить** prefab автоматично (ротація для стін у поточній версії завжди `identity`). Тому кожен sprite вже має бути намальований у правильній орієнтації:

| Варіант | Напрямок малюнка |
|---|---|
| Горизонтальна | Стіна іде ліворуч–праворуч |
| Вертикальна | Стіна іде знизу–вгору |
| Кут NE (правий верхній) | Вигин: знизу–вгору та ліворуч–праворуч з'єднуються у правому верхньому куті |
| Кут NW (лівий верхній) | Вигин у лівому верхньому куті |
| Кут SE (правий нижній) | Вигин у правому нижньому куті |
| Кут SW (лівий нижній) | Вигин у лівому нижньому куті |
| Ворота | Прохід — зазвичай горизонтальний або вертикальний (залежить від арту) |

## Крок 2 — Додати стіну і ворота до реєстру будівель

`BuildingRegistrySO` — головний каталог усіх будівель. Стіна й ворота теж мають бути там зареєстровані як звичайні будівлі.

### 2.1 Відкрити BuildingRegistrySO

1. У **Project** window знайдіть `BuildingRegistrySO` (зазвичай `Assets/Moyva/Settings/`).
2. Клацніть на нього — у **Inspector** з'явиться список `Buildings`.

### 2.2 Додати запис для стіни

1. Натисніть `+` у нижній частині масиву `Buildings`.
2. Заповніть новий елемент:

| Поле | Значення |
|---|---|
| `Id` | `wall` ← **важливо**: точно таке саме значення буде в `WallBuildingId` |
| `Display Name` | `Стіна` |
| `Category` | `Military` (або `Civilian`) |
| `Prefab` | Можна залишити **порожнім** — стіни використовують prefab'и з колекції, а не звідси |

### 2.3 Додати запис для воріт

1. Знову `+` у масиві `Buildings`.
2. Заповніть:

| Поле | Значення |
|---|---|
| `Id` | `gate` ← **важливо**: точно таке саме значення буде в `GateBuildingId` |
| `Display Name` | `Ворота` |
| `Category` | `Military` |
| `Prefab` | Можна залишити **порожнім** |

> **Чому Prefab можна залишити порожнім?**
> Система стін отримує prefab'и з `WallCollectionDefinition`, а не з `BuildingDefinition.Prefab`. Якщо `WallCollectionDefinition.GatePrefab` не призначений, система шукатиме `BuildingDefinition.Prefab` як запасний варіант — але краще призначити prefab саме в колекції.

## Крок 3 — Відкрити Wall Registry Editor

Спеціальне редакторне вікно для зручної роботи з колекціями стін.

1. У верхньому меню Unity: **Moyva → Construction → Wall Registry Editor**.
2. Відкриється вікно `Реєстр стін`.
3. Якщо `BuildingRegistrySO` не знайдено автоматично — натисніть **«Знайти автоматично»** або перетягніть SO в поле зверху.
4. Якщо реєстр ще не створено, натисніть **«Створити новий BuildingRegistrySO»** прямо у вікні.

### Що змінилось у новому редакторі

- Є окремий блок **«Майстер автогенерації зі спрайтів»**.
- Можна передати 7 спрайтів, натиснути одну кнопку і отримати повністю готову колекцію.
- Є прогрес-бар готовності колекції, секції з тултіпами і детальними help box.
- Є масові fallback-операції: для активної колекції і для всіх колекцій одразу.

## Крок 4 — Заповнити колекцію стін

> Є 2 способи: **ручний** (як було раніше) і **автоматичний через Майстер** (рекомендовано).

### 4.0 Рекомендований шлях — автогенерація зі спрайтів

1. У блоці **«Майстер автогенерації зі спрайтів»** оберіть **цільову колекцію**.
2. Вкажіть **папку для prefab'ів** (типово: `Assets/Moyva/Prefabs/Buildings/Walls`).
3. Передайте спрайти:
    - `Горизонтальна`, `Вертикальна`, `NE`, `NW`, `SE`, `SW`, `Ворота`.
4. За потреби налаштуйте опції:
    - створювати `BuildingDefinition`,
    - перезаписувати/не перезаписувати `Prefab` та `Icon` у вже існуючих записах,
    - автозбереження assets.
5. Натисніть **«Згенерувати і сетапити»**.

Після цього інструмент автоматично:
- генерує prefab'и у вказаній папці;
- призначає їх у `WallCollectionDefinition`;
- створює/оновлює `BuildingDefinition` для `WallBuildingId` і `GateBuildingId`;
- застосовує fallback'и до пропущених варіантів.

### 4.0.1 Fallback-логіка в Майстрі

- Якщо не передали `Vertical`, буде використано `Horizontal`.
- Якщо не передали кутові спрайти, вони заповнюються базовим доступним спрайтом.
- Якщо не передали спрайт воріт, `Gate` fallback на `Horizontal`.
- Якщо пропуски лишилися після генерації, можна натиснути:
  - **«Заповнити пропуски fallback'ами»** (лише поточна колекція), або
  - **«Fallback'и для всіх»** (вся таблиця колекцій).

### 4.1 Додати нову колекцію

1. У вікні `Реєстр стін` натисніть **`+ Додати колекцію`**.
2. З'явиться новий рядок `wall-collection-0`.
3. Клацніть на рядок — він розкриється.

### 4.2 Налаштувати ідентифікатори

У розкритому блоці знайдіть секцію **«Ідентифікатори»**:

| Поле | Що заповнити |
|---|---|
| **ID колекції** | Довільна унікальна назва стилю: `stone-wall`, `wooden-wall`, … |
| **ID стіни** | `wall` ← **має збігатися** з `Id` у масиві `Buildings` (Крок 2.2) |
| **ID воріт** | `gate` ← **має збігатися** з `Id` у масиві `Buildings` (Крок 2.3) |

> ⚠️ **Для кількох колекцій** (камінь та дерево одночасно) кожна має свій `WallBuildingId` та `GateBuildingId`:
> - Перша колекція: `WallBuildingId = "stone-wall"`, `GateBuildingId = "stone-gate"`
> - Друга колекція: `WallBuildingId = "wooden-wall"`, `GateBuildingId = "wooden-gate"`
>
> І відповідні записи у `Buildings` у реєстрі.

### 4.3 Призначити prefab'и варіантів стіни

Нижче блок **«Варіанти стіни (6 штук)»** — 6 полів. Для кожного перетягніть відповідний prefab:

| Поле в інспекторі | Який prefab |
|---|---|
| **Горизонтальна ←→** | `Wall_Horizontal.prefab` |
| **Вертикальна ↑↓** | `Wall_Vertical.prefab` |
| **Кут правий верхній — NE ↑→** | `Wall_Corner_NE.prefab` |
| **Кут лівий верхній — NW ↑←** | `Wall_Corner_NW.prefab` |
| **Кут правий нижній — SE ↓→** | `Wall_Corner_SE.prefab` |
| **Кут лівий нижній — SW ↓←** | `Wall_Corner_SW.prefab` |

> Поля без призначеного prefab підсвічуються **червоним** — це підказка що є пробіл.

### 4.4 Призначити prefab воріт

Трохи нижче секція **«Ворота»** — одне поле `Prefab воріт`. Перетягніть `Gate.prefab`.

### 4.5 Перевірити статус

- Якщо біля заголовку колекції є **`✓`** зеленим — всі 7 prefab'ів призначено. Готово.
- Якщо є **`⚠3`** помаранчевим — число показує скільки prefab'ів ще відсутні. Розкрийте колекцію, прокрутіть вниз — там буде список що саме відсутнє.

## Крок 5 — Перевірка у грі

### 5.1 Увійти в режим будівництва

1. Запустіть Play mode.
2. Переключіть GameMode на `Construction` (через UI або хоткей — залежить від вашого налаштування).
3. У меню будівель знайдіть `Стіна` (або `wall` ID) та виберіть її.

### 5.2 Поставити кілька стін

Клацайте на тайли — на кожному з'являється preview:
- **Зелений контур** → можна ставити.
- **Червоний контур** → тайл зайнятий або недоступний.

Натисніть `✓ Підтвердити`. Стіни мають з'явитися на карті.

### 5.3 Перевірити автоматичний вибір варіанта

Поставте дві-три стіни поряд — вони мають автоматично приєднуватися:

```
Поставили: □□□    Очікуваний результат: ├──┤
```

Якщо стіни виглядають однаково (наприклад, всі горизонтальні) але не перетворюються на кути — переходьте до [розділу з помилками](#часті-помилки-та-їх-виправлення).

### 5.4 Поставити ворота

1. Виберіть `Ворота` у меню будівель.
2. Клацніть на тайл, де вже стоїть стіна `wall` — preview має бути зеленим.
3. Підтвердіть. Стіна замінюється воротами.

> ⚠️ Клацання по **порожньому тайлу** при виборі воріт дасть червоний preview — ворота не ставляться поруч зі стіною, а **замість** неї.

### 5.5 Drag-режим

1. Виберіть `Стіна` у меню.
2. Натисніть і **тримайте** палець/мишу на тайлі, де вже є стіна.
3. Тягніть — мають з'являтися preview стін вздовж шляху.
4. Відпустіть → стіни підтверджуються.

## Довідник: який варіант для яких сусідів

Система обирає варіант автоматично за правилом:

| Сусіди (N/E/S/W) | Обраний варіант | Примітка |
|---|---|---|
| Жодного | Горизонтальна | fallback для одиночного сегмента |
| Лише N або лише S | Вертикальна | |
| Лише E або лише W | Горизонтальна | |
| N + S | Вертикальна | пряма вертикаль |
| E + W | Горизонтальна | пряма горизонталь |
| N + E | Кут NE (правий верхній ↑→) | |
| N + W | Кут NW (лівий верхній ↑←) | |
| S + E | Кут SE (правий нижній ↓→) | |
| S + W | Кут SW (лівий нижній ↓←) | |
| N + S + E | Вертикальна | T-подібне, немає окремого варіанта – fallback |
| N + S + W | Вертикальна | T-подібне, fallback |
| E + W + N | Горизонтальна | T-подібне, fallback |
| E + W + S | Горизонтальна | T-подібне, fallback |
| Всі 4 | Горизонтальна | Хрест, окремого варіанта немає – fallback |

> **T-подібні з'єднання та хрести** — якщо в майбутньому з'являться відповідні варіанти спрайтів, достатньо буде додати поля `TJunctionPrefab` і `CrossPrefab` до `WallCollectionDefinition` і оновити `WallPlacementService`. Поточна система готова до розширення.

## Часті помилки та їх виправлення

### ❌ Стіна не з'являється після підтвердження

**Причина:** prefab у `WallCollectionDefinition` не призначено.

**Виправлення:**
1. Відкрийте `Moyva → Construction → Wall Registry Editor`.
2. Знайдіть колекцію з `⚠` — побачите список відсутніх prefab'ів.
3. Призначте prefab'и.

---

### ❌ `ID "wall" не знайдено!` у полі ID стіни

**Причина:** у `BuildingRegistrySO.Buildings` немає запису з `Id = "wall"`.

**Виправлення:**
1. Відкрийте `BuildingRegistrySO` в Inspector.
2. Додайте елемент з `Id = "wall"` (Крок 2.2).
3. Або скористайтесь кнопкою `+ Створити` прямо у вікні Wall Registry Editor — вона створиться автоматично.

---

### ❌ Стіни не оновлюють варіант при будівництві поруч

**Причина:** `ConstructionVisualService` не отримав `IWallPlacementService` через DI.

**Виправлення:**
1. Відкрийте `ConstructionInstaller` у вашій сцені.
2. Переконайтесь, що `WallPlacementService` прив'язаний (`Bind<IWallPlacementService>().To<WallPlacementService>()`).
3. Перевірте, що `BuildingRegistrySO` призначений у Inspector інсталера.

---

### ❌ Ворота можна ставити на порожній тайл

**Причина:** `IConstructionService.TryPreviewAt` не отримує `IWallPlacementService` (injected as `[InjectOptional]`).

**Виправлення:** та само — перевірте прив'язку `IWallPlacementService` в інсталері (див. вище).

---

### ❌ Всі стіни виглядають однаково (не міняють варіант)

**Причина:** `WallBuildingId` у `WallCollectionDefinition` не збігається з `Id` у `BuildingRegistrySO.Buildings`.

**Приклад проблеми:**
- У реєстрі будівель: `Id = "Wall"` (з великої літери)
- У колекції: `WallBuildingId = "wall"` (з малої)

**Виправлення:** привести до однакового написання. ID чутливі до регістру!

---

### ❌ `BuildingRegistrySO не знайдено в проєкті` у Wall Registry Editor

**Причина:** SO ще не створено, або збережено за незвичним шляхом.

**Виправлення:**
1. Правий клік у Project window → `Create → Moyva → Construction → BuildingRegistry`.
2. Збережіть у `Assets/Moyva/Settings/`.
3. Перезапустіть вікно `Wall Registry Editor` або натисніть `Знайти автоматично`.

---

## Підсумок чеклист

Після виконання всіх кроків переконайтесь, що:

- [ ] Створено 7 prefab'ів (6 варіантів стіни + ворота) і збережено в `Assets/`
- [ ] У `BuildingRegistrySO.Buildings` є записи `wall` та `gate`
- [ ] Відкрито `Moyva → Construction → Wall Registry Editor`
- [ ] Додано колекцію, заповнено `CollectionId`, `WallBuildingId = "wall"`, `GateBuildingId = "gate"`
- [ ] Всі 6 полів варіантів стіни мають prefab'и
- [ ] Поле `Prefab воріт` має prefab
- [ ] Біля колекції горить `✓` зеленим
- [ ] Запустили Play mode, поставили стіни — вони з'єднуються автоматично
- [ ] Ворота ставляться лише на тайл зі стіною ✓
