# Графовий редактор — Гайд користувача

← [Назад до Graph System](README.md)

## Огляд інтерфейсу

Редактор відкривається через **Moyva → Graph Editor** або кнопкою "Open Graph Editor" в Inspector ассету `GraphAsset`.

```
┌─────────────────────────────────────────────────────────────┐
│ [Graph: ▼]  Validate  ▶ Run  │ Auto-Layout Group Note │ Mini │ Save │  ← Тулбар
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────┐      ┌──────┐                                     │
│  │Node1├─────►│Node2 │         ← Полотно                   │
│  └─────┘      └──────┘          (GraphView)                │
│                                                             │
│  ┌──────────┐                                              │
│  │ Minimap  │  ← Міні-карта                                │
│  └──────────┘                                              │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ Graph: MyGraph | Nodes: 5 | Connections: 4                  │  ← Статус-бар
└─────────────────────────────────────────────────────────────┘
```

---

## Тулбар — кнопки зліва направо

| Кнопка | Дія |
|---|---|
| **Graph** (ObjectField) | Вибір або перетягування GraphAsset для завантаження |
| **Validate** | Перевіряє граф на помилки: цикли, відсутні з'єднання, несумісні типи |
| **▶ Run** | Запускає граф і показує результат виконання з таймінгом кожного ноду |
| **Auto-Layout** | Автоматично розташовує ноди шарами зліва направо (Kahn's topological sort) |
| **Group** | Групує виділені ноди у візуальну рамку |
| **Note** | Додає липку нотатку (Sticky Note) на полотно |
| **Minimap** (toggle) | Показує/ховає міні-карту в лівому нижньому куті |
| **Save** | Зберігає зміни в GraphAsset (= `EditorUtility.SetDirty + AssetDatabase.SaveAssets`) |

## Створення нодів

**Правий клік** на порожньому місці полотна відкриває вікно пошуку нодів.

Ноди згруповані за категоріями:
- **Generators** — джерела даних (HeightSource, BiomeResolver, Voronoi, Fertility)
- **Converters** — перетворення типів (HeightToTile)
- **Processing** — обробка карт (Smooth, Erosion, Terrace, NoiseCombiner, CellularAutomata)
- **Terrain** — ландшафтні операції (SeaCoastline, AutoTileTransition)
- **Features** — особливості світу (Lake, Forest, River, Mountain, POI, Road)
- **Utility** — допоміжні (Mask, BorderFrame, Overlay, TileReplace, MapScale)
- **Logic** — логіка (ConditionalSwitch)
- **Analysis** — аналіз (ChokepointAnalyzer)
- **WFC** — Wave Function Collapse
- **Core** — Output
- **Debug** — DebugPreview

Ви також можете **почати вводити назву** ноду для фільтрації.

## З'єднання портів

1. **Наведіть** курсор на вихідний порт (кружечок праворуч від ноду)
2. **Натисніть і тягніть** до вхідного порту (кружечок зліва від іншого ноду)
3. Відпустіть — з'єднання створено

**Правила з'єднань:**
- Можна з'єднувати тільки **Output → Input** (не Input → Input)
- Типи мають бути **сумісні** (зелений ↔ зелений, блакитний ↔ блакитний)
- Один Input-порт приймає **тільки одне** з'єднання (нове замінює старе)
- Один Output-порт може мати **кілька** з'єднань

**Видалення з'єднання:** Натисніть на лінію з'єднання → натисніть `Delete`.

---

## Кольори портів

Кольори допомагають миттєво зрозуміти, який тип даних передається:

| Колір | Тип | Приклад використання |
|---|---|---|
| 🟢 **Зелений** | `float[,]` — HeightMap | Карти висот, fertility, noise |
| 🔵 **Блакитний** | `string[,]` — TileMap | Карти біомів, тайлів, об'єктів |
| 🟡 **Жовтий** | `bool[,]` — Mask | Маски (водна, лісова, чокпоінт) |
| 🟠 **Оранжевий** | `int[,]` — IntMap | POI карта, регіони Вороного |
| ⚪ **Сірий** | `int` | Скалярні значення (ширина, висота) |
| 💚 **Світло-зелений** | `float` | Скалярні float-значення |
| 🟣 **Фіолетовий** | `Vector2Int` | Координати |

> **Підказка:** Наведіть курсор на порт — з'явиться тултіп з назвою та типом даних, наприклад: `"HeightMap (HeightMap)"` або `"Mask (Mask)"`.

## Валідація графа

Кнопка **"Validate"** перевіряє:

| Перевірка | Що означає помилка |
|---|---|
| **Null Nodes** | Вузол з відсутнім скриптом (зламана серіалізація) |
| **Cycles** | Граф містить цикл — це заборонено! Нод A → B → C → A |
| **Invalid Connections** | З'єднання вказує на неіснуючий вузол або порт |
| **Type Mismatch** | З'єднані порти з несумісними типами |
| **Unconnected Inputs** | Input-порт не має з'єднання (Warning, не Error) |

При помилках — детальний лог виводиться в Unity Console.

## Запуск графа (▶ Run)

Кнопка **"▶ Run"** виконує граф прямо в редакторі:

1. **Валідація** — якщо є помилки, виконання відміняється
2. **Виконання** — кожен нод виконується послідовно (MapSize за замовчуванням: 64×64)
3. **Результат** — статус-бар показує:
   - `✓ Run completed in 150ms | 8 nodes executed | Total node time: 142.5ms`
4. **Підсвічування нодів:**
   - 🟢 Зелена рамка — успішно
   - 🟡 Жовта рамка — warning
   - 🔴 Червона рамка — error
   - 🟠 Оранжева рамка — повільний (>100ms)
5. **Тултіп** — наведіть на нод, щоб побачити його час виконання

> **Зверніть увагу:** Run у редакторі використовує фіксований MapSize 64×64 та випадковий seed. Для повноцінного тестування запустіть гру через Play Mode.

## Міні-карта

Міні-карта знаходиться в **лівому верхньому** куті. Вона показує зменшену версію всього полотна з усіма нодами.

- **Натискання** на міні-карті — переміщує вигляд
- **Toggle** — ховає/показує через кнопку "Minimap" у тулбарі

---

## Auto-Layout

Кнопка **"Auto-Layout"** автоматично розташовує ноди:

- Використовує **алгоритм Кана** для визначення "шарів" графа
- Ноди без залежностей — зліва, фінальний Output — справа
- Відстань між шарами: 300px горизонтально, 120px вертикально
- Ноди центруються вертикально в кожному шарі

> Корисно після імпорту або коли граф став заплутаним.

---

## Groups (групи)

Групи — це візуальні рамки, що об'єднують кілька нодів:

1. **Виділіть** кілька нодів (тягніть прямокутник або Shift+Click)
2. Натисніть **"Group"** або **Ctrl+G**
3. З'явиться рамка з написом "Node Group"
4. **Подвійний клік** на заголовку — перейменуйте (наприклад, "Terrain Generation")
5. Перетягування групи переміщує всі ноди всередині

---

## Sticky Notes (нотатки)

Нотатки — текстові блоки прямо на полотні:

1. Натисніть **"Note"** або **Ctrl+T**
2. З'явиться блок з текстом "..."
3. **Подвійний клік** — редагуйте заголовок і текст
4. Перетягніть і масштабуйте як потрібно

Використовуйте для:
- Пояснення складних ділянок графа
- TODO-нотаток
- Документації логіки генерації

---

## Inspector (інспектор нодів)

Натисніть **лівою кнопкою** на нод — його параметри відобразяться в Unity Inspector:

- Всі `[SerializeField]` поля ноду (threshold, tile name, noise settings...)
- ScriptableObject поля (наприклад, `DataNoiseSettings`) — розгортаються inline, без необхідності шукати окремий ассет
- Зміни в Inspector моментально зберігаються в GraphAsset

> **Важливо:** Не змінюйте поля `NodeId` та `EditorPosition` вручну — вони керуються системою автоматично.

---

## Play Mode та Read-Only

Коли ви запускаєте гру в Unity (Play Mode):

- Редактор автоматично переключається в **Read-Only** режим
- Не можна створювати, видаляти або переміщувати ноди
- Статус-бар показує ` | READ-ONLY (Play Mode)`
- При виході з Play Mode — редактор відновлюється

---

## Persistence (збереження стану)

Редактор **пережива** domain reload, recompile та Play Mode:

- GraphAsset зберігається за GUID
- При перезавантаженні Unity — граф відновлюється автоматично
- Позиція та масштаб полотна зберігаються

## Copy/Paste нодів

| Дія | Що відбувається |
|---|---|
| **Ctrl+C** | Зберігає виділені ноди в буфер (з усіма параметрами) |
| **Ctrl+V** | Створює нові ноди з тими ж параметрами, але новими ID, зі зміщенням +40px |
| **Ctrl+D** | Комбінація Copy+Paste в один крок |

Буфер — **глобальний** між усіма графами (можна скопіювати ноди з одного графа і вставити в інший).

> **Зверніть увагу:** З'єднання між нодами НЕ копіюються — тільки самі ноди з їхніми параметрами.
