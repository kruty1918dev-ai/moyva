# Довідник нодів: Обробка (Processing)

← [Назад до Graph System](README.md)

---

Ноди обробки **трансформують** існуючі карти висот. Вони приймають HeightMap і повертають модифіковану HeightMap. Зазвичай ставляться **між** генератором (HeightSource) та конвертером (HeightToTile).

Типова ланка: `HeightSource → Smooth → Terrace → Erosion → HeightToTile`

---

## Noise Combiner

**Категорія:** Processing · **Файл:** `NoiseCombinerNode.cs`

Комбінує **дві карти висот** в одну за допомогою математичної операції. Ідеально для створення складного рельєфу з простих компонентів.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap A | `float[,]` | Перша карта висот |
| **Input** 🟢 | HeightMap B | `float[,]` | Друга карта висот |
| **Output** 🟢 | Combined | `float[,]` | Результат комбінування |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Mode** | `CombineMode` | Add | Режим комбінування |
| **Blend** | `float` | 0.5 | Вага змішування (тільки для режиму Lerp: 0.0 = повністю A, 1.0 = повністю B) |

### Режими

| Режим | Формула | Коли використовувати |
|---|---|---|
| **Add** | `Clamp01(A + B)` | Додати висоти з двох джерел (з обмеженням 0–1) |
| **Multiply** | `A × B` | Маскування — "прибрати" висоти де B низьке |
| **Max** | `max(A, B)` | Залишити вище з двох |
| **Min** | `min(A, B)` | Залишити нижче з двох |
| **Lerp** | `lerp(A, B, blend)` | Плавне змішування двох карт |

> **Примітка:** Якщо карти мають різний розмір, менша карта доповнюється нулями.

### Приклад

```
HeightSource(Scale:30)  ──┐
                          ├──► NoiseCombiner(Lerp, 0.3) ──► HeightToTile
HeightSource(Scale:100) ──┘

Результат: Великі пагорби (Scale:100) з дрібними деталями (Scale:30, вага 0.3)
```

## Smooth

**Категорія:** Processing · **Файл:** `SmoothNode.cs`

Згладжує карту висот **box blur** фільтром. Прибирає різкі перепади, створюючи плавні пагорби.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Вхідна карта висот |
| **Output** 🟢 | SmoothedMap | `float[,]` | Згладжена карта |

### Параметри

| Параметр | Тип | Діапазон | За замовч. | Опис |
|---|---|---|---|---|
| **Radius** | `int` | 1–5 | 1 | Радіус згладжування в тайлах |
| **Iterations** | `int` | 1–10 | 1 | Кількість проходів |

### Як працює

Для кожної клітинки обчислюється **середнє** значення всіх сусідів у квадраті (2*radius+1) × (2*radius+1):

```
Radius 1 (3×3):         Radius 2 (5×5):
┌───┬───┬───┐          ┌───┬───┬───┬───┬───┐
│ N │ N │ N │          │ N │ N │ N │ N │ N │
├───┼───┼───┤          ├───┼───┼───┼───┼───┤
│ N │ X │ N │          │ N │ N │ N │ N │ N │
├───┼───┼───┤          ├───┼───┼───┼───┼───┤
│ N │ N │ N │          │ N │ N │ X │ N │ N │
└───┴───┴───┘          ├───┼───┼───┼───┼───┤
9 сусідів              │ N │ N │ N │ N │ N │
                       ├───┼───┼───┼───┼───┤
                       │ N │ N │ N │ N │ N │
                       └───┴───┴───┴───┴───┘
                       25 сусідів
```

### Поради

- **Radius 1, Iterations 1** — мінімальне згладжування, залишає деталі
- **Radius 2, Iterations 3** — середнє, створює плавні пагорби
- **Radius 3+, Iterations 5+** — сильне, перетворює в майже плоску поверхню
- В ітераційному режимі — кожна ітерація використовує результат попередньої

## Terrace

**Категорія:** Processing · **Файл:** `TerraceNode.cs`

Створює **терасовий ефект** — розбиває плавний градієнт висоти на дискретні рівні (як рисові тераси або геологічні плато).

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Вхідна карта висот |
| **Output** 🟢 | TerracedMap | `float[,]` | Карта з терасовим ефектом |

### Параметри

| Параметр | Тип | Діапазон | За замовч. | Опис |
|---|---|---|---|---|
| **Levels** | `int` | 2–20 | 5 | Кількість рівнів терасування |
| **Sharpness** | `float` | 0.0–1.0 | 0.8 | Різкість переходу (0=плавно, 1=різко) |

### Як це працює

```
Вхід (плавний градієнт):       Вихід (5 рівнів, Sharp=1.0):
1.0 ┤                  ╱       1.0 ┤              ┌──────
    │                ╱             │              │
0.8 ┤              ╱           0.8 ┤         ┌────┘
    │            ╱                 │         │
0.6 ┤          ╱               0.6 ┤    ┌────┘
    │        ╱                     │    │
0.4 ┤      ╱                   0.4 ┤────┘
    │    ╱                         │
0.2 ┤  ╱                       0.2 ┤────
    │╱                             │
0.0 └────────────────          0.0 └──────────────
```

### Формула

```csharp
level = floor(height * levels) / levels        // квантизація
lerp(height, level, sharpness)                 // інтерполяція для плавності
```

### Ігрове застосування

- **5 рівнів** — створює чіткі "рівні висоти" для стратегічної карти
- **3 рівні** — суша/пагорби/гори
- **Sharpness 0.5** — м'які переходи між терасами

## Erosion

**Категорія:** Processing · **Файл:** `ErosionNode.cs`

Симулює **термальну ерозію** — матеріал "осипається" з крутих схилів до сусідніх клітинок, створюючи реалістичний рельєф.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Вхідна карта висот |
| **Output** 🟢 | ErodedMap | `float[,]` | Ерозована карта |

### Параметри

| Параметр | Тип | Діапазон | За замовч. | Опис |
|---|---|---|---|---|
| **Erosion Rate** | `float` | 0.001–0.1 | 0.01 | Швидкість ерозії — скільки матеріалу переноситься за крок |
| **Iterations** | `int` | 1–50 | 10 | Кількість ітерацій ерозії |
| **Talus Angle** | `float` | 0.0–1.0 | 0.05 | Максимальний кут спокою — мінімальна різниця висот для початку ерозії |

### Як працює

```
На кожній ітерації, для кожної клітинки:

  1. Знайти найнижчого сусіда
  2. Різниця = height[x,y] - height[neighbor]
  3. Якщо різниця > talusAngle:
     → Перемістити половину різниці від поточної клітинки до сусіда

До ерозії:        Після 10 ітерацій:      Після 30 ітерацій:
   ╱\                  ╱‾‾\                   ╱‾‾‾‾‾\
  ╱  \                ╱    \                 ╱       ‾\
 ╱    \              ╱      ‾\              ╱          ‾\
╱      \            ╱         ‾\           ╱             ‾\_
Гострий пік        Згладжений        Плоска вершина з пологими схилами
```

### Поради

- **Talus Angle 0.01** — реалістична ерозія для гір
- **Talus Angle 0.05** — мінімальна ерозія, зберігає рельєф
- **Iterations 20+** — сильна ерозія, створює плоскі долини з пологими схилами
- Комбінуйте з **Smooth** для максимально реалістичного рельєфу

## Cellular Automata

**Категорія:** Processing · **Файл:** `CellularAutomataNode.cs`

Запускає **клітинний автомат** на булевій мапі (true/false). Ідеальний для генерації печер, островів, або органічних форм.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟡 | InputMask | `bool[,]` | Початкова маска (наприклад, від RandomScatter або Mask) |
| **Output** 🟡 | OutputMask | `bool[,]` | Оброблена маска |

### Параметри

| Параметр | Тип | Діапазон | За замовч. | Опис |
|---|---|---|---|---|
| **Iterations** | `int` | 1–20 | 4 | Кількість кроків автомата |
| **Birth Threshold** | `int` | 0–8 | 5 | Мінімум живих сусідів для "народження" мертвої клітинки |
| **Survival Threshold** | `int` | 0–8 | 4 | Мінімум живих сусідів для виживання живої клітинки |

### Як працює

Використовує **Moore neighbourhood** — 8 сусідніх клітинок:

```
┌───┬───┬───┐
│ N │ N │ N │
├───┼───┼───┤       Для кожної клітинки:
│ N │ X │ N │       neighbors = рахуємо "живих" (true) з 8 сусідів
├───┼───┼───┤
│ N │ N │ N │       Якщо X мертва і neighbors >= birthThreshold → X стає живою
└───┴───┴───┘       Якщо X жива і neighbors < survivalThreshold → X помирає
```

### Приклад: генерація печер

```
Початок (50% random):    Після 5 ітерацій:
░░█░░█░█░░█░░░█░        ████████████░░████
░█░██░░░█░░█░░░█        ████████████░░████
█░░░██░░█░█░░██░   →    ████░░░█████░░████
░░█░░█░░░█░░█░░█        ████░░░░████░░████
░░░█░█░██░█░█░░░        ████░░░░████░░████
                         ████████████░░████

█ = стіна (true), ░ = печера (false)
```

### Типові рецепти

| Рецепт | Birth | Survival | Ітерації | Результат |
|---|---|---|---|---|
| **Печери** | 5 | 4 | 5 | Органічні порожнини з'єднані тунелями |
| **Острови** | 6 | 5 | 3 | Круглі суцільні ділянки в "морі" |
| **Рями (болота)** | 4 | 3 | 2 | Розкидані дрібні групи |
