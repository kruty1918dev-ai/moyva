# Швидкий старт — Перший граф за 5 хвилин

← [Назад до Graph System](README.md)

---

## Крок 1: Створити GraphAsset

1. У Unity натисніть **правою кнопкою** в Project window
2. Оберіть **Create → Moyva → Generator → Graph Asset**
3. Назвіть файл, наприклад `MyFirstGraph`

> **Що таке GraphAsset?** — Це ScriptableObject, який зберігає ваш граф: усі ноди та з'єднання між ними. Він живе як файл в папці Assets.

---

## Крок 2: Відкрити графовий редактор

Є два способи:

**Спосіб 1:** Виділіть `MyFirstGraph` в Project → в Inspector натисніть кнопку **"Open Graph Editor"**

**Спосіб 2:** Через меню Unity: **Moyva → Graph Editor**, потім перетягніть ваш GraphAsset у поле "Graph" в тулбарі

Відкриється вікно з темним полотном і сіткою — це ваш графовий редактор.

---

## Крок 3: Створити ноди

1. **Натисніть правою кнопкою** на порожньому полотні
2. Відкриється вікно пошуку з категоріями

Створіть ці ноди по черзі:

### 3.1 — Height Source (джерело висот)
- В пошуку знайдіть **"Height Source"** (категорія Generators)
- Натисніть на нього — нод з'явиться на полотні
- **Цей нод генерує карту висот** — основу всього ландшафту

### 3.2 — Height To Tile (висоти → тайли)
- Створіть нод **"Height To Tile"** (категорія Converters)
- **Цей нод перетворює числові висоти в назви тайлів** (наприклад, висота 0.7 → `"mountain"`)

### 3.3 — Output (вихід)
- Створіть нод **"Output"** (категорія Core)
- **Це фінальний нод** — він збирає результат генерації

---

## Крок 4: З'єднати ноди

Тепер з'єднайте їх:

1. **Перетягніть** від зеленого output-порту `HeightMap` на **Height Source** до зеленого input-порту `HeightMap` на **Height To Tile**
2. **Перетягніть** від блакитного output-порту `TileMap` на **Height To Tile** до блакитного input-порту `BiomeMap` на **Output**
3. **Перетягніть** від зеленого output-порту `HeightMap` на **Height Source** до зеленого input-порту `HeightMap` на **Output**

Ваш граф має виглядати так:

```
┌──────────────┐     ┌──────────────┐     ┌──────────┐
│ Height Source ├────►│ Height→Tile  ├────►│  Output  │
│              ├────────────────────────►  │          │
└──────────────┘     └──────────────┘     └──────────┘
   HeightMap ─────────► HeightMap            BiomeMap
                                             HeightMap
```

> **Підказка:** Порти одного кольору — сумісні. Зелений (HeightMap) можна з'єднати тільки з зеленим, блакитний (TileMap) — з блакитним.

---

## Крок 5: Налаштувати параметри

1. **Натисніть на нод** `Height Source` — він виділиться
2. Подивіться в Unity **Inspector** (справа) — там відобразяться параметри вузла
3. В полі **Noise Settings** призначте або створіть `DataNoiseSettings` ScriptableObject
4. Налаштуйте аналогічно **Height Map Settings** для `Height To Tile`

> **Не бачите параметри?** Переконайтесь, що натиснули саме на нод (не на полотно). В Inspector має відобразитися назва ноду.

---

## Крок 6: Валідація

Натисніть кнопку **"Validate"** в тулбарі редактора.

- ✅ **"Validation passed"** — все гаразд!
- ❌ **Помилки** — перевірте, чи всі порти з'єднані та чи налаштовані ScriptableObject конфіги

---

## Крок 7: Запустити граф

Натисніть кнопку **"▶ Run"** в тулбарі.

Що відбудеться:
1. Система перевірить граф (валідація)
2. Виконає кожен нод по порядку
3. В статус-барі внизу з'явиться повідомлення: `✓ Run completed in Xms | N nodes executed`
4. Ноди підсвітяться кольором:
   - 🟢 **Зелений** — виконано успішно
   - 🟡 **Жовтий** — є попередження
   - 🟠 **Оранжевий** — повільний нод (>100мс)
   - 🔴 **Червоний** — помилка

---

## Крок 8: Зберегти

Натисніть **"Save"** або `Ctrl+S`.

---

## Що далі?

Тепер, коли базовий граф працює, додайте більше нодів:

| Хочу... | Додайте нод |
|---|---|
| Згладити рельєф | [Smooth](nodes-processing.md#smooth) між HeightSource та HeightToTile |
| Додати море | [Sea/Coastline](nodes-terrain.md#sea--coastline) після HeightToTile |
| Додати озера | [Lake Generation](nodes-features.md#lake-generation) |
| Посадити ліси | [Forest Cluster](nodes-features.md#forest-cluster) |
| Розмістити села | [POI Placement](nodes-features.md#poi-placement) |
| Побудувати дороги | [Road/Path](nodes-features.md#road--path) між селами |

Кожен нод детально описаний у відповідних розділах довідника.

## Як підключити до гри

В `GeneratorInstaller` (файл `Assets/Moyva/Scripts/Features/Generator/Runtime/GeneratorInstaller.cs`):

1. Увімкніть чекбокс **"Use Graph Generator"**
2. Призначте ваш `GraphAsset` у поле
3. Запустіть гру — `GraphBasedMapDataGenerator` виконає граф замість старого pipeline
