## Height Source

**Категорія:** Generators · **Файл:** `HeightSourceNode.cs`

Головне джерело даних для будь-якого графа. Генерує карту висот (`float[,]`) через алгоритм Perlin noise.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Output** 🟢 | HeightMap | `float[,]` | Згенерована карта висот (0.0–1.0) |

### Параметри

| Параметр | Тип | Опис |
|---|---|---|
| **Noise Settings** | `DataNoiseSettings` (SO) | ScriptableObject з налаштуваннями шуму |

### Як налаштувати `DataNoiseSettings`

| Поле | Що робить | Типове значення |
|---|---|---|
| `Scale` | Масштаб шуму. Більше значення → більші, плавніші пагорби | 50 |
| `Octaves` | Кількість "шарів" деталізації. Більше → детальніший рельєф | 4 |
| `Persistance` | Вплив кожної наступної октави (0–1). Вище → більше дрібних деталей | 0.5 |
| `Lacunarity` | Множник частоти між октавами. Вище → більше дрібних варіацій | 2.0 |
| `Seed` | Зерно генерації. Однаковий seed → однакова карта | 0 |
| `Offset` | Зміщення карти по X/Y | (0, 0) |

### Як це виглядає

```
Scale: 30, Octaves: 1         Scale: 50, Octaves: 4
┌────────────────┐            ┌────────────────┐
│░░▒▒▓▓████▓▓▒▒░│            │░▒░▓█▒░▓▒█▓░▒▓▒│
│░░▒▒▓▓████▓▓▒▒░│            │▒▓█▓▒░▓█▓▒░▓█▒░│
│░░▒▒▓▓████▓▓▒▒░│  Плавно    │░▒▓▒█▓▒░▓█▒▓░▒█│  Детально
│░░▒▒▓▓████▓▓▒▒░│            │▓░▒█▒▓░▒▓█▒░▓▒░│
└────────────────┘            └────────────────┘
```

> **Порада:** Почніть з Scale 40–60, Octaves 3–4, Persistance 0.45–0.55 для природного вигляду.

## Biome Resolver

**Категорія:** Generators · **Файл:** `BiomeResolverNode.cs`

Призначає **біоми** (типи тайлів) на основі висоти та вологості. Використовує зовнішній сервіс `IBiomeResolver`, який додатково генерує карту вологості через Perlin noise.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Карта висот |
| **Input** 🔵 | BaseTileMap | `string[,]` | Базова карта тайлів (буде модифікована) |
| **Output** 🔵 | BiomeMap | `string[,]` | Результуюча карта біомів |

### Параметри

| Параметр | Тип | Опис |
|---|---|---|
| **Biomes Settings** | `DataBiomesSettings` (SO) | Масив правил біомів з діапазонами висоти/вологості |

### Як працює

```
Для кожної клітинки:
  height = heightMap[x, y]        → наприклад, 0.45
  moisture = noise(x, y)          → наприклад, 0.7

  Перебір BiomeData[]:
    "desert":    height 0.3–0.5, moisture 0.0–0.3  → НЕ підходить (moisture 0.7 > 0.3)
    "grassland": height 0.3–0.6, moisture 0.4–0.8  → ПІДХОДИТЬ! → biomeMap[x,y] = "grassland"
```

### Типовий результат

| Висота ↓ / Вологість → | 0.0–0.3 | 0.3–0.6 | 0.6–1.0 |
|---|---|---|---|
| **0.0–0.2** (низовини) | `"sand"` | `"swamp"` | `"swamp"` |
| **0.2–0.5** (рівнини) | `"desert"` | `"grassland"` | `"forest"` |
| **0.5–0.8** (пагорби) | `"rock"` | `"hills"` | `"mountain_forest"` |
| **0.8–1.0** (гори) | `"snow"` | `"snow"` | `"snow"` |

## Fertility Map

**Категорія:** Generators · **Файл:** `FertilityMapNode.cs`

Генерує **карту родючості** — показує, де на карті земля найбільш придатна для рослинності та сільського господарства. Враховує Perlin noise, оптимальну висоту та близькість до води.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Карта висот |
| **Input** 🟡 | WaterMask | `bool[,]` | Маска водних клітинок |
| **Output** 🟢 | FertilityMap | `float[,]` | Значення родючості 0.0–1.0 для кожної клітинки |
| **Output** 🟡 | FertileMask | `bool[,]` | true де fertility > 0.5 |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Noise Settings** | `DataNoiseSettings` | — | Шум для базової варіації родючості |
| **Optimal Min Height** | `float` | 0.25 | Мінімальна "оптимальна" висота |
| **Optimal Max Height** | `float` | 0.55 | Максимальна "оптимальна" висота |
| **Height Influence** | `float` | 0.4 | Наскільки сильно висота впливає (0 = не впливає, 1 = повний вплив) |
| **Water Proximity Bonus** | `float` | 0.3 | Бонус родючості біля води |
| **Water Bonus Radius** | `int` | 5 | Відстань в тайлах, на яку поширюється водний бонус |

### Формула

```
fertility = baseNoise * (1 - heightInfluence)
          + heightBonus * heightInfluence        ← бонус якщо висота в оптимальному діапазоні
          + waterBonus                            ← бонус за близькість до води

де heightBonus = 1.0 якщо висота всередині діапазону, плавно спадає до 0 на краях
   waterBonus  = waterProximityBonus * (1 - dist/radius) якщо вода ближче за radius
```

### Ігрове застосування

- **Родюча земля** (зелені долини біля річок) → хороше місце для сіл
- **Неродюча земля** (сухі пагорби, далеко від води) → складне виживання
- Створює **емерджентний геймплей**: розташування сіл визначається ландшафтом

> **Порада:** Подайте `WaterMask` від `Lake Generation` або `Sea/Coastline` щоб врахувати всі водойми.

## Voronoi Regions

**Категорія:** Generators · **Файл:** `VoronoiRegionsNode.cs`

Генерує **діаграму Вороного** — розбиває карту на регіони, кожен з яких "належить" найближчій зерновій точці. Корисний для створення провінцій, територій або як базова структура для біомів.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** ⚪ | MapWidth | `int` | Ширина карти |
| **Input** ⚪ | MapHeight | `int` | Висота карти |
| **Output** 🟠 | RegionIds | `int[,]` | ID регіону для кожної клітинки (0, 1, 2, ...) |
| **Output** 🟢 | DistanceMap | `float[,]` | Відстань до найближчої зернової точки (0.0–1.0) |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Region Count** | `int` | 12 | Кількість регіонів (2–100) |
| **Seed** | `int` | 0 | Зерно для позицій точок |
| **Distance Map** | `bool` | false | Чи обчислювати карту відстаней |

### Як це виглядає

```
Region Count: 5                Region Count: 20
┌────────────────┐            ┌────────────────┐
│1111112222233333│            │12345678901234567│
│1111122222233333│            │12344567890123456│
│1111122222233333│  5 великих │11234567890123456│  20 дрібних
│4444442222255555│  регіонів  │11223456789012345│  регіонів
│4444444455555555│            │11223345678901234│
└────────────────┘            └────────────────┘
```

### Ігрове застосування

- **Провінції на карті** — кожен регіон = окрема територія
- **Різні біоми** — комбінуйте з `ConditionalSwitch` для призначення різних тайлів кожному регіону
- **DistanceMap** — використовуйте як маску для плавних переходів на кордонах

## Height To Tile

**Категорія:** Converters · **Файл:** `HeightToTileNode.cs`

Конвертує числову карту висот (`float[,]`) в текстову карту тайлів (`string[,]`), присвоюючи кожному діапазону висот свій ID тайлу.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Карта висот |
| **Output** 🔵 | TileMap | `string[,]` | Карта ID тайлів |

### Параметри

| Параметр | Тип | Опис |
|---|---|---|
| **Height Map Settings** | `HeightMapSettings` (SO) | Масив шарів висоти (HeightLayer[]) |

### Як налаштувати `HeightMapSettings`

```
HeightLayer[]:
  ┌─────────────────────────────────────────┐
  │ [0] TileID: "water_deep"  MaxHeight: 0.2│  ← 0.0–0.2 → water_deep
  │ [1] TileID: "sand"        MaxHeight: 0.3│  ← 0.2–0.3 → sand
  │ [2] TileID: "grass"       MaxHeight: 0.5│  ← 0.3–0.5 → grass
  │ [3] TileID: "hills"       MaxHeight: 0.7│  ← 0.5–0.7 → hills
  │ [4] TileID: "mountain"    MaxHeight: 1.0│  ← 0.7–1.0 → mountain
  └─────────────────────────────────────────┘
```

> **Різниця з BiomeResolver:** HeightToTile враховує тільки висоту. BiomeResolver додатково враховує вологість для складніших біомів.
