# Довідник нодів: Терейн (Terrain)

← [Назад до Graph System](README.md)

---

Ноди терейну **формують ландшафтні елементи** великого масштабу — береги, автоматичні переходи між різними типами тайлів.

---

## Sea / Coastline

**Категорія:** Terrain · **Файл:** `SeaCoastlineNode.cs`

Генерує **морську берегову лінію** з трьома зонами: глибоке море, мілководдя та пляж. Берег може бути з будь-якого боку карти.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Карта висот (для noise-модуляції берега) |
| **Input** 🔵 | TileMap | `string[,]` | Базова карта тайлів (буде модифікована) |
| **Output** 🔵 | TileMap | `string[,]` | Карта з морем та береговою лінією |
| **Output** 🟡 | WaterMask | `bool[,]` | Маска водних тайлів (true = вода) |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Coast Side** | `enum` | South | Бік карти: North, South, East, West |
| **Base Depth** | `int` | 8 | Глибина зони моря в тайлах |
| **Coast Width** | `int` | 3 | Ширина прибережної зони в тайлах |
| **Beach Width** | `int` | 2 | Ширина пляжної зони в тайлах |
| **Noise Influence** | `float` | 0.3 | Вплив HeightMap на форму берега (0 = рівний, 1 = дуже нерівний) |
| **Deep Sea Tile** | `string` | `"water_deep"` | ID тайлу глибокого моря |
| **Shallow Tile** | `string` | `"water_shallow"` | ID мілководдя |
| **Beach Tile** | `string` | `"sand"` | ID пляжного тайлу |

### Зони берега

```
Coast Side: South

 ┌────────────────────────────────────┐
 │ grass  grass  grass  grass  grass │  ← Суша (не змінюється)
 │ grass  grass  grass  grass  grass │
 ├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
 │ sand   sand   sand   sand   sand  │  ← Beach (beachWidth: 2)
 │ sand   sand   sand   sand   sand  │
 ├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
 │ w_shal w_shal w_shal w_shal w_sh │  ← Shallow (coastWidth: 3)
 │ w_shal w_shal w_shal w_shal w_sh │
 │ w_shal w_shal w_shal w_shal w_sh │
 ├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
 │ w_deep w_deep w_deep w_deep w_de │  ← Deep sea (baseDepth: 8)
 │ w_deep w_deep w_deep w_deep w_de │
 │ ...                               │
 └────────────────────────────────────┘
```

### Noise Influence

З HeightMap = 0 (рівний берег):
```
════════════════════
```

З HeightMap модуляцією 0.3:
```
══╗  ╔═══╗  ╔══════
  ╚══╝   ╚══╝        ← лінія берега "дихає" з ландшафтом
```

> **WaterMask** — використовуйте як вхід для `FertilityMap`, `ForestCluster` та інших нодів, що враховують водні зони.

## AutoTile Transitions

**Категорія:** Terrain · **Файл:** `AutoTileTransitionNode.cs`

Автоматично **визначає переходи між різними типами тайлів**. Аналізує кожну клітинку та її 8 сусідів, підставляючи правильний суфікс (`_cliff_n`, `_corner_ne`, тощо) на основі бітової маски.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🔵 | TileMap | `string[,]` | Карта тайлів (без переходів) |
| **Input** 🟢 | HeightMap | `float[,]` | Карта висот (для визначення "рівнів") |
| **Output** 🔵 | TileMap | `string[,]` | Карта з суфіксами переходів |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Height Tier Step** | `float` | 0.2 | Крок рівня висоти — визначає, що вважається "різним рівнем" |

### Як працює — бітова маска

Для кожної клітинки перевіряються 8 сусідів:

```
Біти:  128  1   2
        64 [X]  4
        32  16  8

Якщо сусід належить іншому "рівню висоти" → відповідний біт = 1
```

### Приклади суфіксів

| Біти сусідів (вище/нижче рівня) | Суфікс | Візуально |
|---|---|---|
| Тільки North (1) | `_cliff_n` | Обрив зверху |
| North + East (1+4=5) | `_cliff_ne` | Обрив з двох боків |
| NW + N + NE (128+1+2=131) | `_cliff_n` | Повна стіна зверху |
| Тільки NE кут (2) | `_corner_ne` | Кутовий елемент |
| Нічого | без суфікса | Звичайний тайл |

### Приклад

```
До (звичайна карта):          Після (з автотайлами):
┌─────┬─────┬─────┐          ┌──────────┬──────────┬──────────┐
│grass│grass│grass│          │grass     │grass     │grass     │
├─────┼─────┼─────┤          ├──────────┼──────────┼──────────┤
│grass│mount│mount│    →     │grass_cl_e│mount_cl_w│mount     │
├─────┼─────┼─────┤          ├──────────┼──────────┼──────────┤
│grass│grass│mount│          │grass     │grass_cr_ │mount_cl_s│
└─────┴─────┴─────┘          └──────────┴──────────┴──────────┘
```

### Ігрове застосування

- **Автоматичні клифи** — гірський тайл поруч з рівниною автоматично отримує відповідний вигляд
- **Берегові лінії** — водний тайл поруч з сушею → правильний візуальний перехід
- Дуже корисний **після HeightToTile** — перетворює "блочну" карту в природну

> **Важливо:** Ваші тайли (спрайти/сети) мають містити відповідні варіанти з суфіксами: `grass_cliff_n`, `grass_cliff_ne`, `grass_corner_ne` тощо.
