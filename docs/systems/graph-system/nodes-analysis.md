## Chokepoint Analyzer

**Категорія:** Analysis · **Файл:** `ChokepointAnalyzerNode.cs`

Аналізує карту для знаходження **вузьких проходів** (chokepoints) — місць, де прохід обмежений з двох боків перешкодами (гори, вода). Ці точки мають стратегічне значення для оборони та контролю.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🟢 | HeightMap | `float[,]` | Карта висот |
| **Input** 🟡 | BarrierMask | `bool[,]` | Маска перешкод (true = стіна/вода/гора) |
| **Output** 🟢 | DefenseScore | `float[,]` | Оцінка оборонного значення (0.0–1.0) |
| **Output** 🟡 | ChokepointMask | `bool[,]` | true де виявлено вузький прохід |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Min Passage Width** | `int` | 2 | Мінімальна ширина проходу для визначення чокпоінта |
| **Max Passage Width** | `int` | 6 | Максимальна ширина (ширше = не чокпоінт) |
| **Search Directions** | `int` | 4 | Кількість напрямків перевірки (4 або 8) |

### Алгоритм

```
Для кожної вільної клітинки (де barrierMask = false):

1. BFS Distance Transform — відстань до найближчої перешкоди
   distToBarrier[x,y] = мін. відстань до будь-якої клітинки де barrier = true

2. Для кожного напрямку (N-S, E-W, і діагональні):
   - Знайти відстань до перешкоди з обох боків
   - passageWidth = leftDist + rightDist

3. Якщо minPassageWidth ≤ passageWidth ≤ maxPassageWidth:
   → Це чокпоінт!
   → defenseScore = 1.0 - (passageWidth - minPassageWidth) / (maxPassageWidth - minPassageWidth)
```

### Defense Score

| Значення | Ігровий сенс |
|---|---|
| **0.9–1.0** | Ідеальний чокпоінт — 1-2 тайли ширини. Замок чи вежі тут дуже ефективні |
| **0.5–0.8** | Вузький прохід — 3-4 тайли. Добра позиція для оборони |
| **0.1–0.4** | Помірно обмежений прохід. Тактична перевага, але не повна блокада |
| **0.0** | Відкрита місцевість — немає оборонної переваги |

### Ігрове застосування

- **AI оборона:** NPC розміщують вежі/гарнізони на чокпоінтах
- **Quest system:** "Захистіть вузький прохід від ворога"
- **Стратегічна цінність:** Території з чокпоінтами цінніші
- **POI Placement:** Подайте `DefenseScore` у POI для розміщення фортець
- **Світобудування:** Замки історично будувались в чокпоінтах

### Типова ланка

```
HeightSource → Mask(>0.7) ──── BarrierMask
                    │
                    ├───► ChokepointAnalyzer
                    │              │
HeightSource  ──────┘         DefenseScore → (для AI/POI)
                              ChokepointMask → (для візуалізації)
```

> **Порада:** Комбінуйте `WaterMask` та `Mask(>0.7)` через `NoiseCombiner` або логіку, щоб отримати повну маску перешкод (вода + гори).
