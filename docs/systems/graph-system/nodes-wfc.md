# Довідник нодів: Wave Function Collapse (WFC)

← [Назад до Graph System](README.md)

---

**Wave Function Collapse** — алгоритм процедурної генерації, що створює карти на основі зразка (прикладу). Замість ручних правил, WFC "вчиться" з невеликого зразкового зображення і генерує велику карту, що виглядає подібно.

---

## WFC Generator

**Категорія:** WFC · **Файл:** `WaveFunctionCollapseNode.cs`

Реалізує **Overlapping Model WFC** — бере зразкову тайлову карту і генерує нову карту будь-якого розміру, зберігаючи локальні патерни.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🔵 | SampleMap | `string[,]` | Зразкова карта (невеликий приклад бажаного результату) |
| **Output** 🔵 | GeneratedMap | `string[,]` | Згенерована карта |

### Параметри

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| **Pattern Size** | `int` | 3 | Розмір патерну NxN (2 або 3) |
| **Output Width** | `int` | 32 | Ширина результуючої карти |
| **Output Height** | `int` | 32 | Висота результуючої карти |
| **Symmetry** | `int` | 8 | Кількість варіантів обертання/відображення (1, 2, 4 або 8) |
| **Max Attempts** | `int` | 10 | Макс. спроб при суперечності |
| **Ground** | `bool` | false | Чи фіксувати нижній ряд за зразком |

### Інтерфейси

- **`IAsyncNode`** — підтримує асинхронне виконання (не блокує UI)
- **`IPreviewableNode`** — може показувати preview в редакторі

### Алгоритм (WFCAlgorithm.cs)

WFC працює в чотири етапи:

#### 1. Витягування патернів

З вхідного зразка витягаються всі NxN патерни з їхніми частотами:

```
Зразок 5×5:           Патерни 3×3 (з обертанням):
┌─┬─┬─┬─┬─┐           ┌─┬─┬─┐  ┌─┬─┬─┐  ┌─┬─┬─┐
│G│G│W│G│G│           │G│G│W│  │G│W│G│  │W│G│G│  ...
│G│W│W│W│G│    →      │G│W│W│  │W│W│W│  │W│W│G│  
│G│G│W│G│G│           │G│G│W│  │G│W│G│  │W│G│G│
│G│G│G│G│G│           └─┴─┴─┘  └─┴─┴─┘  └─┴─┴─┘
│G│G│G│G│G│           + 90°, 180°, 270° + відображення = до 8 варіантів кожного
└─┴─┴─┴─┴─┘

G = grass, W = water
```

#### 2. Ініціалізація

Кожна клітинка вихідної карти починає з **усіма можливими патернами** (суперпозиція):

```
Клітинка [3,5]: {pattern_0, pattern_1, pattern_2, ..., pattern_47}
                  ↑ всі 47 можливих патернів
```

#### 3. Collapse (згортання)

Повторюємо поки всі клітинки не вирішені:

1. **Observe** — знайти клітинку з найменшою **Shannon entropy** (найменша кількість можливих патернів, зважених на логарифм частот)
2. **Collapse** — вибрати один патерн для цієї клітинки (зважений випадковий вибір за частотами)
3. **Propagate** — оновити сусідні клітинки (видалити несумісні патерни)

```
До:    [48 можливих]  [48]  [48]  [48]
       [48]           [48]  [48]  [48]

Після collapse [0,0] = "grass-grass-water":
       [1 вибрано]    [12]← зменшилось  [48]  [48]
       [15]←          [30]←              [48]  [48]
```

#### 4. Суперечності та повтори

Якщо після propagation якась клітинка лишилась з 0 можливих патернів → **суперечність**. Алгоритм починає з початку (до `maxAttempts` спроб).

### Constraint Propagation

Використовує **стек** (не рекурсію) для ефективної propagation:

```
Stack<(x, y)> propagationStack;
while (stack.Count > 0) {
    var cell = stack.Pop();
    for each neighbor:
        if (removed any patterns from neighbor)
            stack.Push(neighbor);
}
```

### Коли використовувати

| Сценарій | WFC підходить? |
|---|---|
| Потрібна карта "як на зразку" | ✅ Ідеально |
| Потрібні лише патерни | ✅ Добре |
| Потрібен контроль висоти/біомів | ❌ Краще HeightSource + BiomeResolver |
| Великі карти (100×100+) | ⚠️ Повільно, краще для маленьких ділянок |
| Підземелля / лабіринти | ✅ Відмінно з правильним зразком |

## Constraint Polish

**Категорія:** WFC · **Файл:** `ConstraintPolishNode.cs`

**Полірує** карту тайлів за допомогою WFC-подібних правил сусідства. На відміну від повного WFC, не генерує з нуля — натомість виправляє "нелегальні" комбінації сусідніх тайлів.

### Порти

| Напрямок | Назва | Тип | Опис |
|---|---|---|---|
| **Input** 🔵 | TileMap | `string[,]` | Карта тайлів (можливо з невірними переходами) |
| **Output** 🔵 | TileMap | `string[,]` | Полірована карта |

### Як працює

1. Отримує правила сусідства від `IWFCService` (зареєстрованого через Zenject)
2. Для кожної клітинки перевіряє: чи допущений поточний тайл поруч з усіма сусідами?
3. Якщо ні — замінює на найкращий дозволений тайл

### Коли використовувати

- **Після ручних модифікацій** — якщо інші ноди створили невалідні комбінації тайлів
- **Після генерації** — для виправлення артефактів на стиках різних ландшафтних зон
- Ставьте **перед** `AutoTileTransition` і **перед** `Output`

### Типова ланка

```
... → Sea/Coastline → Forest Cluster → ConstraintPolish → AutoTile → Output
```
