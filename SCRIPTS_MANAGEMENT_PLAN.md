## 📝 ЯК ЗМІНЮВАТИ СКРИПТИ

### ЯКЩО ПОТРІБЕН РЕФАКТОРИНГ
1. **Ідентифікуйте модуль** з таблиці вище
2. **Виділіть клас** з >500 строк
3. **Розбийте** на 2-3 менших класи
4. **Оновіть** всі посилання (refs)
5. **Запустіть тести** в модулі
6. **Коміт**: `refactor: split {ClassName} into smaller parts`

### ЯКЩО ПОТРІБНА ОПТИМІЗАЦІЯ
1. **Профайл** за допомогою Unity Profiler
2. **Знайдіть bottleneck**
3. **Оптимізуйте** алгоритм або дані
4. **Перевірте** швидкість
5. **Коміт**: `perf: optimize {FeatureName} by {improvement}%`

### ЯКЩО ПОТРІБНА НОВА ФІЧА
1. **Виберіть** відповідний модуль
2. **Додайте** інтерфейс в API/
3. **Реалізуйте** в Runtime/
4. **Напишіть** тести в Tests/
5. **Коміт**: `feat({Module}): add {FeatureName}`

### ЯКЩО ПОТРІБНО ВИДАЛИТИ КОД
1. **Перевірте** использання (Ctrl+K Ctrl+R в VS Code)
2. **Видаліть** посилання
3. **Видаліть** файл
4. **Запустіть** тести
5. **Коміт**: `refactor: remove dead code {FileName}`

## 🎓 НАВЧАЛЬНИЙ МАТЕРІАЛ

### Ключові патерни проекту:
1. **API + Runtime розділення** - інтерфейси в API/, реалізація в Runtime/
2. **Zenject DI** - Dependency Injection контейнер для об'єктів
3. **Signal Bus** - Pub/Sub evento система для слабого зв'язку
4. **ScriptableObjects** - Конфігурація та дані без hardcode
5. **Graph Nodes** - Generator використовує граф вузлів для гнучкості

### Новачкам:
- Почніть з малих модулів (Animations, Clouds, Pathfinding)
- Розберіться в API → Runtime шаблоні
- Напишіть перший тест в Economy модулі
- Прочитайте документацію в Features/{Module}/README.md (якщо існує)
